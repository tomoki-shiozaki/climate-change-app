# フロントエンド CONTRIBUTING ガイド

このドキュメントでは、フロントエンド開発における CSS/UI の方針をまとめています。

---

## CSS・UI 方針

### 1. Tailwind CSS を基本とする

- React プロジェクトでは Tailwind CSS をメインで使用します。
- ユーティリティクラスを使って、シンプルなスタイル付けを行います。
- Tailwind の知識が十分でない場合でも、既存コードを参考にスタイルを追加してください。

### 2. 複雑なコンポーネント

- 複雑な UI（ナビゲーションバー、モーダルなど）は UI ライブラリを使用して構築しても構いません。
- 当面は [react-bootstrap](https://react-bootstrap.github.io/) を使用します。
- shadcn-ui は現状では使用しません。

### 3. カスタマイズ方針

- なるべく抽象化されているコンポーネントや規約に従ったスタイルを使用してください。
- 過度なカスタマイズは避け、コードの可読性・保守性を優先します。

### 4. デザインの考え方

- デザインの美しさよりも、ユーザーに分かりやすく、使いやすいことを優先します。
- Tailwind を基本にして、必要に応じて react-bootstrap のスタイルを併用します。

---

この方針に従うことで、フロントエンドコードの一貫性を保ち、将来的なメンテナンスを容易にします。
